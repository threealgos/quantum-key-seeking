Do you need an API token / login?Yes, to submit jobs to Helios (H1-1E emulator, H2-1 hardware, etc.) through the qnexus Python SDK.
You do not manually create or copy-paste an "API token" like with IBM Quantum (no export IBMQ_TOKEN=... style).
Instead, you authenticate once (or every ~30 days) using your Quantinuum Nexus credentials (email + password or Microsoft account).
This login creates and stores short-lived access/refresh tokens automatically on your machine (in ~/.qnexus/ or similar cache).

How to get access and authenticateCreate / get a Quantinuum Nexus account (free for basic access, paid/approved for real hardware credits):Go to: https://nexus.quantinuum.com/
Sign up or log in (supports email/password or Microsoft account).
Accept terms & conditions after registration.
For real hardware (Helios H2-1 etc.), you may need to apply for access / credits via Quantinuum (academic/research often approved; check their portal or contact support@quantinuum.com).
Emulator (H1-1E) is usually immediately available after signup.

Authenticate in your Python environment (this is what your code needs):After pip install qnexus, run one of these once (or when token expires):Option A – Easiest (opens browser for login)
In terminal or Jupyter:python

import qnexus as qnx
qnx.login()               # Opens browser → log in to nexus.quantinuum.com → done

Option B – Headless / script-friendly (username + password)  python

import qnexus as qnx
qnx.login_with_credentials(username="your@email.com", password="your_password")

Option C – If already logged in via browser / JupyterHub
In Quantinuum's hosted Nexus Jupyter environment, authentication is silent/automatic — no extra step.

Verify it's working:python

import qnexus as qnx
print(qnx.is_authenticated())  # Should print True
print(qnx.Account.limits())    # Shows your shot/job quotas

Your code already has:python

if not qnx.is_authenticated():
    qnx.login()

So it will prompt you automatically when needed (browser login preferred).
Token storage & expiration:Tokens stored locally (~30 days validity).
When expired: re-run qnx.login() (or code will prompt).
No need for manual export QNEXUS_API_TOKEN=... — unlike older Quantinuum APIs or IBM.
